## Project purpose

Web interface for controlling a Raspberry Pi camera (`rpicam-still`) for astrophotography. The UI runs on the Pi and is accessed via a browser on the local network.

## Commands

```bash
npm run dev      # Vite dev server (frontend only, no Express backend)
npm run astro    # Express server on :3000 serving the built dist/
npm run build    # Build frontend into dist/
npm run lint     # ESLint
```

For full-stack development (API + UI), build first then run `npm run astro`. The Vite dev server (`npm run dev`) cannot proxy to the Express API in its current configuration (ViteExpress is commented out).

## Architecture

**Two-tier, single-origin**: React SPA (Vite) + Express backend. In production, Express serves `dist/` as static files and exposes REST routes under `/api/`.

**Backend routes** (`server/`):
- `GET /api/project/list` — reads `dist/projects/` directory
- `POST /api/project/add` — creates `dist/projects/<name>/` and `dist/projects/<name>/preview/`
- `GET /api/capture/preview?command=<cmd>` — executes the given shell command via `execSync` and returns stdout as base64
- `GET /api/capture/capture?command=<cmd>` — same but returns `{ok: true}`

Projects are persisted as directories in `dist/projects/` (inside the build output). This means running `npm run build` wipes and recreates `dist/`, which would delete existing project folders.

**Frontend state** (`src/`):
- `App.jsx` holds the active `project` string (empty = show `ProjectBrowser`, set = show `Project`)
- `ProjectBrowser` lists and creates projects via API
- `Project` renders an Accordion with "Focus and Target" (`Preview`) and "Exposure" (`FileTest`) panels
- `Preview` builds and sends `rpicam-still ... --output -| base64` commands; receives base64 JPEG and renders it via `Histogram`
- `Histogram` draws the image on a `<canvas>` and overlays RGB channel histograms plus a Laplacian variance sharpness score
- `FileTest` is a local-file version of the histogram tool (no camera API call)

**shadcn/ui**: Components in `src/components/ui/` are generated by shadcn (style: new-york, no TSX). Add new ones with `npx shadcn@latest add <component>`. Do not manually restructure these files.

**Path alias**: `@` resolves to `src/` (configured in `vite.config.js` and `jsconfig.json`).

**Tailwind**: v4 via `@tailwindcss/vite` plugin. Global CSS entry is `src/styles/globals.css`.
